2.ThreadLocal

ThreadLocal并不是一个Thread，而是Thread的局部变量。 ThreadLocal为每个线程提供单独一份存储空间，具有线程隔离的效果，只有在线程内才能获取到对应的值，线程外则不能访问。

在被问simpledateformat线程不安全怎么处理的时候，也可以用这个Threadlocal

可以理解为，某个事件的参与者 拉了个私群，方便内部及时交流

ThreadLocal常用方法:

  • public void set(T value)设置当前线程的线程局部变量的值
  • public T get()返回当前线程所对应的线程局部变量的值
  • .public void remove()移除当前线程的线程局部变量

1.2 方法名

  • Result 返回结果类

这是一个带泛形的类

    • 这是一个通用的后端统一返回结果类 Result，用于封装后端处理请求后的响应数据。它的主要作用是标准化接口的返回格式，使得前端可以统一处理成功和失败的情况
    • implements Serializable：表示该类可以被序列化（例如，用于网络传输或缓存）。

  • EmployeeLoginVO

返回的格式而已

  • @RequestBody EmployeeLoginDTO employeeLoginDTO
    • @RequestBody 的作用 功能：将 HTTP 请求的 请求体（Body） 中的 JSON 或 XML 数据，自动转换为 Java 对象。
    • 通过 @RequestBody 将 JSON 数据绑定到 EmployeeLoginDTO 对象。
    • 这里返回的username=admin password=123456

​ 这是一个返回值类型为Result的（里面规范为EmployeeLoginVO）名称为login的方法，将登录请求发送的json格式的数据转换为employeeLoginDTO类型的变量
